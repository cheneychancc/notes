# 🛡️ 前端安全核心体系（面试版）

> 👉 核心就四个：**XSS / CSRF / Token / HTTPS**

---

# 🔥 一、XSS（跨站脚本攻击）

## 🧠 是什么

> 攻击者把恶意 JS 注入页面执行

---

## 💥 例子

```html
<input value="<script>alert(1)</script>" />
```

```js
div.innerHTML = userInput;
```

👉 执行脚本 ❗

---

## 🧨 危害

* 偷 cookie
* 冒充用户
* 控制页面

---

## 🛡️ 防御（重点背）

### ✅ 1. HTML 转义（最核心）

```js
textContent / innerText
```

或：

```js
&lt; &gt; &amp;
```

---

### ✅ 2. 不用 innerHTML

👉 改用：

```js
element.textContent = xxx;
```

---

### ✅ 3. CSP（内容安全策略）

```http
Content-Security-Policy: script-src 'self'
```

---

## 🎯 面试一句话

> XSS 是把用户输入当代码执行，防御核心是“永远不要相信用户输入”，要做 HTML 转义或使用安全 API。

---

# 🔥 二、CSRF（跨站请求伪造）

## 🧠 是什么

> 👉 利用浏览器“自动带 cookie”，伪造请求

---

## 💥 例子

用户登录了银行：

```text
cookie: session=abc
```

攻击者网站：

```html
<img src="https://bank.com/transfer?money=1000">
```

👉 浏览器自动带 cookie，钱被转走 ❗

---

## 🛡️ 防御（重点）

### ✅ 1. CSRF Token（最常用）

```js
请求头带 token
```

服务器校验

---

### ✅ 2. SameSite Cookie

```http
Set-Cookie: SameSite=Strict
```

👉 跨站不带 cookie

---

### ✅ 3. Referer / Origin 校验

---

## 🎯 面试一句话

> CSRF 利用了浏览器自动携带 cookie 的机制，防御核心是“让请求不可伪造”，通常通过 CSRF Token 或 SameSite Cookie 实现。

---

# 🔥 三、Token vs Cookie（高频对比🔥）

## 🧠 Cookie

* 浏览器自动带
* 存在 cookie 里
* 易受 CSRF

---

## 🧠 Token（JWT）

```js
Authorization: Bearer xxx
```

* 手动带
* 存 localStorage / memory
* 不会自动发送 → 防 CSRF

---

## ⚠️ Token 也有问题

👉 容易被 XSS 偷！

---

## 🧠 最佳实践

| 风险   | 解决                                  |
| ---- | ----------------------------------- |
| XSS  | 不存 localStorage / 用 httpOnly cookie |
| CSRF | SameSite / Token                    |

---

## 🎯 面试一句话

> Cookie 适合服务端 session，但有 CSRF 风险；Token 更适合前后端分离，但需要防 XSS。

---

# 🔥 四、HTTPS（传输安全）

## 🧠 是什么

> 👉 HTTP + TLS 加密

---

## 🔐 解决什么问题？

* 防窃听（加密）
* 防篡改（完整性）
* 防伪造（证书）

---

## 🧠 简单流程（面试够用）

1. 客户端发起请求
2. 服务端返回证书
3. 客户端验证证书
4. 协商对称密钥
5. 后续通信加密

---

## 🎯 面试一句话

> HTTPS 通过非对称加密交换密钥，再用对称加密传输数据，保证安全和性能。

---

# 🔥 五、补充高频点（加分🔥）

---

## 1️⃣ 点击劫持（Clickjacking）

```html
<iframe src="xxx"></iframe>
```

👉 诱导用户点击

### 防御：

```http
X-Frame-Options: DENY
```

---

## 2️⃣ SQL 注入（了解即可）

👉 后端问题，但你可以说：

> 前端要做输入校验，但核心防御在后端

---

## 3️⃣ 敏感信息存储

❌ localStorage（容易被 XSS 拿）

✅ httpOnly cookie

---

# 🧠 六、终极总结（面试王炸）

你可以直接这样说👇

---

> 前端安全主要包括 XSS 和 CSRF：
>
> * XSS 是注入脚本攻击，防御核心是对用户输入做转义，避免 innerHTML
> * CSRF 是利用 cookie 自动发送伪造请求，防御核心是 Token 或 SameSite
>
> 在认证方面：
>
> * Cookie 有 CSRF 风险
> * Token 有 XSS 风险
>
> HTTPS 则解决传输安全问题，防止数据被窃听和篡改

