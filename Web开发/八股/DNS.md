面试官问 DNS，其实是在考察你对**网络协议栈**和**网络安全**的深度理解。DNS 不仅仅是域名变 IP，它还涉及到缓存机制和分布式架构。

---

### 一、 DNS 解析的全过程：从“查询”到“带路”

当你在浏览器输入 `www.google.com` 时，背后经历了一个**递归**与**迭代**相结合的过程：

1.  **浏览器缓存**：浏览器先看自己有没有记录。
2.  **系统缓存 (Hosts)**：浏览器没有，就问操作系统（比如 Windows 的 `hosts` 文件）。
3.  **路由器缓存**：还没找到，就问你家的路由器。
4.  **ISP 本地 DNS 服务器 (LDNS)**：这是你的运营商（电信/移动）提供的服务器。**绝大多数解析到这里就结束了**，因为 LDNS 缓存了绝大部分常用域名。
5.  **根域名服务器 (Root Server)**：如果 LDNS 也没有，它会去问“根”。根服务器不直接告诉你 IP，但它告诉你：`.com` 区域由谁管。
6.  **顶级域名服务器 (TLD)**：LDNS 再去问 `.com` 服务器。它告诉你：`google.com` 是由哪个权威服务器管。
7.  **权威域名服务器 (Authoritative Server)**：这是最终站。LDNS 问它，它返回 `www` 对应的真实 IP 地址。
8.  **返回并缓存**：LDNS 把结果给浏览器，并自己存一份（根据 TTL 时间），方便下次秒开。

### 二、 DNS 劫持 (DNS Hijacking) —— “带错路”

**本质：** 攻击者强行控制了你的 DNS 服务器，或者篡改了你的配置。

* **手段：**
    * **本地攻击**：木马病毒修改了你电脑的 `hosts` 文件。
    * **路由攻击**：黑客攻破了你家路由器的管理后台，修改了 DNS 设置。
    * **运营商拦截**：某些小运营商为了省流量或打广告，拦截了你的 DNS 请求，强行让你跳转到一个充满广告的页面。
* **现象：** 你明明输入的是 A 网站，浏览器地址栏也显示 A 网站，但页面内容却是 B 网站（甚至由于地址没变，这种攻击隐蔽性极高）。

---

### 三、 DNS 污染 (DNS Cache Poisoning) —— “给假情报”

**本质：** 攻击者在 DNS 服务器还没来得及返回正确答案前，**抢先一步**发回一个伪造的错误 IP。

* **原理：**
    * 由于 DNS 查询通常使用 **UDP 协议**（无连接，不验证身份），黑客只要在公网上监听到你的 DNS 请求，就立刻伪装成 DNS 服务器给你回一个“查无此人”或“错误 IP”。
    * 因为 UDP 是先入为主的，你的电脑收到第一个回复（假的）就会采纳，后面那个真的回复会被直接丢弃。
* **现象：** 表现为网站打不开（显示链接重置），或者解析到了一个完全不存在的 IP。这常见于某些敏感网站的屏蔽手段。



---

### 四、 💡 进阶：如何防御？（面试亮点）

作为前端或全栈开发者，你应该知道现代互联网是如何对抗这两者的：

1.  **HTTPDNS**：
    * **方案**：绕过系统原生的 DNS 解析逻辑，直接发起一个 **HTTP 请求**去问服务器 IP。
    * **优点**：因为 HTTP 是基于 TCP 的，且可以自建加密，彻底解决了运营商劫持和污染问题。在大厂（如腾讯、阿里）的移动端 App 中是标配。
2.  **DoH (DNS over HTTPS) / DoT (DNS over TLS)**：
    * **原理**：将 DNS 查询包加密封装在 HTTPS/TLS 隧道里。
    * **效果**：中间人看不见你查了什么，也无法伪造回复。
3.  **DNSSEC**：
    * 给 DNS 记录加上数字签名，确保你收到的回复真的是权威服务器发的，且没被改过。

### 🎤 面试总结话术

> “DNS 解析是一个**层层递归**的过程，核心是为了实现域名到 IP 的高效转化。
>
> **DNS 劫持**侧重于‘路径控制’，即控制了你的查询终端或服务器；而 **DNS 污染**侧重于‘抢答伪造’，利用 UDP 协议的缺陷注入假数据。
>
> 在我之前的项目中，为了保证海外用户访问的稳定性，我们会关注 **TTL 的设置**。针对劫持风险较高的场景，我们会建议使用 **HTTPDNS** 技术。这不仅能防劫持，还能结合地理位置实现更精准的 **调度分发（GSLB）**，让用户访问到物理距离最近的服务器。”

**既然聊到了网络安全，面试官可能会顺着问你：“如果 DNS 解析出的 IP 是对的，但内容还是被篡改了，那是什么攻击？”（答案是 HTTP 内容劫持，需要靠 HTTPS 解决）。你对 HTTPS 的握手过程熟悉吗？**
