## Web Workers

是一种可以运行在 Web 应用程序后台线程，独立于主线程之外的技术。

JavaScript 语言是单线程模型的，而通过使用 Web Workers，我们可以创造多线程环境，从而可以发挥现代计算机的多核 CPU 能力，在应对规模越来越大的 Web 程序时也有较多收益。

## 引入项目

需要考虑

1. 兼容性？
> 2011年已支持大多数主流浏览器
2. 使用场景？
> 1.`Worker API`的局限性：同源限制，无DOM对象，异步通信，适合不涉及DOM操作的任务。
> 2.`Worker`的使用成本：创建时间+数据传输时间。可以预创建，可以忽略创建时间，只考虑数据传输成本。
> 3.任务特点：需要是可并行的多任务，为了充分利用多核能力，可并行的任务数越接近CPU数量，收益越高。

>[!TIP]
>结论：可并行的计算密集型任务适合用 Worker 来做。
